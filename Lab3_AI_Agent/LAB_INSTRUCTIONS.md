# Lab 3: AI Security Agent

## Overview

Build an autonomous AI-powered SOC agent that:

1. **Subscribes** to network packet events via Pub/Sub (push).
2. **Analyzes** each event with Gemini 2.5 Flash using Function Calling.
3. **Blocks** malicious IPs automatically via VPC Firewall.
4. **Logs** every decision (blocked/allowed + reasoning) to Firestore.
5. **Generates** a CISO-grade Markdown dashboard on demand.

### Architecture

```
VM (vm_publisher.py)
        |
        v
  [ Pub/Sub Topic: packet-logs-topic ]
        |  (Push Subscription)
        v
  [ Cloud Run: ai-sec-agent ]
        |
   +---------+-----------+
   |         |           |
   v         v           v
Gemini   VPC Firewall  Firestore
2.5 Flash (block IP)  (event log)
```

## Deployment Steps

> **ORDER MATTERS.** Follow these steps exactly.

### Step 1: Run Infrastructure Setup

```bash
cd Lab3_AI_Agent
chmod +x setup_infra.sh
./setup_infra.sh
```

This creates: Service Account, Pub/Sub topic, and enables all required APIs. (VPC Firewall deny rules are created dynamically by the agent at runtime.)

### Step 2: Initialize Firestore (MANUAL)

> This step **cannot** be automated via `gcloud` — it must be done in the Console.

1. Go to: https://console.cloud.google.com/firestore
2. Click **Create Database**
3. Select **Native mode**
4. Location: `asia-southeast1`
5. Click **Create**

### Step 3: Build and Deploy Cloud Run

```bash
cd agent

export PROJECT_ID=$(gcloud config get-value project)
export REGION=asia-southeast1

# Build the container image
gcloud builds submit --tag gcr.io/$PROJECT_ID/ai-sec-agent .

# Deploy to Cloud Run
gcloud run deploy ai-sec-agent \
    --image gcr.io/$PROJECT_ID/ai-sec-agent \
    --region $REGION \
    --service-account ai-sec-agent-sa@$PROJECT_ID.iam.gserviceaccount.com \
    --set-env-vars GOOGLE_CLOUD_PROJECT=$PROJECT_ID \
    --allow-unauthenticated \
    --port 8080
```

> **WARNING:** The `--allow-unauthenticated` flag is **REQUIRED** for this lab.
> Without it, the Pub/Sub push subscription will receive **403 Forbidden** errors
> because Cloud Run defaults to requiring authentication.
>
> In production, you would instead configure a **Push Auth Service Account**
> with `roles/run.invoker` on the Pub/Sub subscription. For this lab,
> unauthenticated access is used for simplicity.

Save the **Service URL** from the deployment output — you need it in the next step.

### Step 4: Create Pub/Sub Push Subscription

> This step MUST come AFTER Cloud Run is deployed (you need the Service URL).

```bash
SERVICE_URL=$(gcloud run services describe ai-sec-agent \
    --region $REGION \
    --format 'value(status.url)')

gcloud pubsub subscriptions create packet-logs-sub \
    --topic packet-logs-topic \
    --push-endpoint "${SERVICE_URL}/pubsub" \
    --ack-deadline 60
```

### Step 5: Run the Publisher

```bash
cd ..  # back to Lab3_AI_Agent/

pip install google-cloud-pubsub
export GOOGLE_CLOUD_PROJECT=$(gcloud config get-value project)

# Mock mode (no root required)
python vm_publisher.py

# Or with live capture (requires root + scapy)
# sudo python vm_publisher.py --sniff
```

You should see logs like:

```
2025-01-01 12:00:00 [INFO] Published: TCP 172.16.0.9 -> 10.0.0.1 [severity=critical, msgID=123456]
```

### Step 6: View the Cloud Run Dashboard (curl)

```bash
curl "${SERVICE_URL}/dashboard"
```

This returns a Markdown-formatted CISO Executive Summary generated by Gemini.

### Step 7: Launch the Streamlit Dashboard (optional)

For a full web UI with metrics, interactive tables, and AI analysis:

```bash
cd dashboard
pip install -r requirements.txt
export GOOGLE_CLOUD_PROJECT=$(gcloud config get-value project)

streamlit run app.py
```

Open http://localhost:8501 in your browser. Click **Refresh & Analyze** in the sidebar.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Pub/Sub push returns **403** | Cloud Run requires authentication | Redeploy with `--allow-unauthenticated` or add Push Auth SA with `roles/run.invoker` |
| `json.dumps` crashes with `TypeError: datetime` | Firestore returns `datetime` objects | Already handled in code via `serialize_doc()` — if you see this, check the `/dashboard` endpoint |
| Firewall rule insert fails with **already exists** | Same IP was blocked before (`ai-block-*` rule exists) | Expected — the agent handles this gracefully and logs "already blocked" |
| Firestore query fails | Database not initialized | Complete Step 2 — Firestore must be in **Native mode** |
| `vm_publisher.py` fails with permission error | Missing `roles/pubsub.publisher` | Re-run `setup_infra.sh` or manually add the role |
| Cloud Run cold start timeout | First request takes >10s | Increase Pub/Sub `--ack-deadline` (currently 60s, should be sufficient) |

## Cleanup

To avoid ongoing charges after the lab:

```bash
# Delete Cloud Run service
gcloud run services delete ai-sec-agent --region $REGION --quiet

# Delete Pub/Sub subscription and topic
gcloud pubsub subscriptions delete packet-logs-sub --quiet
gcloud pubsub topics delete packet-logs-topic --quiet

# Delete AI-created firewall rules
gcloud compute firewall-rules list --filter="name~'^ai-block-'" --format="value(name)" \
    | xargs -I {} gcloud compute firewall-rules delete {} --quiet

# Delete Service Account
gcloud iam service-accounts delete ai-sec-agent-sa@$PROJECT_ID.iam.gserviceaccount.com --quiet

# Delete Firestore data (optional — via Console)
```
